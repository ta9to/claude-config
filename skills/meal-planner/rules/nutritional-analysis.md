## 分析の目的

ユーザーが報告した直近2〜3日の食事から、**不足していると思われる栄養素・食材カテゴリ**を特定し、献立選定に反映させる。

詳細な栄養計算は不要。「このグループが足りなそう」という判断で十分。

---

## 食材カテゴリと主な栄養素

| カテゴリ | 主な栄養素 | 不足サインの例 |
|----------|-----------|--------------|
| 野菜（緑黄色・淡色） | ビタミン、食物繊維、ミネラル | 「コンビニ飯続き」「外食ばかり」 |
| たんぱく質（肉・魚・卵・豆） | タンパク質、鉄、亜鉛 | 「麺類ばかり」「炭水化物多め」 |
| 発酵食品（味噌・ヨーグルト・納豆） | 善玉菌、腸内環境 | 毎日の食事に発酵食品がない |
| 乳製品 | カルシウム | 乳製品の言及がない |
| 主食（米・パン・麺） | 炭水化物・エネルギー | 特に問題になりにくいが過多に注意 |
| 果物 | ビタミンC、抗酸化物質 | 果物の言及がない |

---

## 偏りパターンと対応

### 外食・コンビニ多め
→ **野菜不足・塩分過多**が多い
→ 野菜中心のメニュー（野菜炒め、具沢山の味噌汁、サラダ添え）を優先

### 麺・パン・丼ばかり
→ **野菜・タンパク質不足**
→ 肉や魚を使ったメインに野菜を組み合わせるメニューを選ぶ

### 肉ばかり
→ **食物繊維・ビタミン不足**
→ 魚料理を1食以上入れ、副菜に野菜を必ず加える

### 自炊・和食中心
→ バランスが比較的良い傾向
→ ユーザーの気分を優先して献立を組む

### 疲れ・胃もたれを訴えている
→ **消化に良いもの**を優先
→ 煮物・蒸し料理・雑炊・うどん・豆腐を含むメニューを選ぶ

## 分析結果の伝え方

ユーザーへのフィードバックは**シンプルな一言**で添える。長い栄養解説は不要。

良い例：
```
直近は外食が多かったとのことで、野菜をしっかり摂れるメニューを多めに入れてみました。
```

```
お肉が続いていたようなので、お魚を1日入れつつ、副菜は野菜多めで組んでみます。
```

悪い例（過剰な解説）：
```
ビタミンCが不足しており、鉄の吸収率を上げるためにも…（長い説明）
```

---

## 優先度ルール

1. **アレルギー・苦手食材** → 絶対に除外（最優先）
2. **気分・食べたいもの** → 栄養より優先してよい（継続できる食事が大事）
3. **栄養バランス** → 気分を考慮した上で調整する
4. **コスト・調理時間** → 現実的な範囲で最適化する
